## Installation

### Quick Start (One Command Setup) ğŸš€

**For the fastest setup, use the all-in-one command:**

```bash
# 1. Install dependencies
npm install

# 2. Create .env file (copy from .env.example or create manually)
# See Environment Variables section below

# 3. Run everything in one command (creates DB + tables + starts server)
npm run start:full

# Or for development mode with auto-reload:
npm run dev:full

# To also seed demo data automatically:
npm run start:seed
# or
npm run dev:seed
```

The `start:full` command automatically:
- âœ… Checks for `.env` file
- âœ… Builds CSS (if needed)
- âœ… Creates database (if doesn't exist)
- âœ… Creates all tables (if don't exist)
- âœ… Optionally seeds demo data (with `--seed` flag)
- âœ… Starts the server

### Manual Setup (Step by Step)

1. **Clone or navigate to the project directory**
   ```bash
   cd stella-pet-services
   ```

2. **Install dependencies**
   ```bash
   npm install
   ```

3. **Set up environment variables**
   Create a `.env` file in the project root:
   ```bash
   DB_HOST_STAGE=localhost
   DB_PORT_STAGE=3306
   DB_NAME_STAGE=stella_pet_services
   DB_USER_STAGE=root
   DB_PASSWORD_STAGE=your_password
   SESSION_SECRET=your-secret-key-here
   PORT=3000
   NODE_ENV=development
   ```

4. **Create database and tables**
   ```bash
   npm run migrate
   ```

5. **Seed demo data (optional)**
   ```bash
   npm run seed
   ```

6. **Create admin user (optional)**
   ```bash
   npm run create-admin admin@example.com yourpassword
   ```

7. **Build TailwindCSS**
   ```bash
   npm run build:css:prod
   ```
   
   Or for development with watch mode:
   ```bash
   npm run build:css
   ```
   (Run this in a separate terminal to watch for CSS changes)

8. **Start the development server**
   ```bash
   npm run dev
   ```
   
   Or for production:
   ```bash
   npm start
   ```

9. **Open your browser**
   ```
   http://localhost:3000
   ```

## Available Scripts

### Main Commands
- `npm start` - Start the production server
- `npm run dev` - Start development server with nodemon (auto-restart)
- `npm run start:full` - **One-command setup**: Creates DB + tables + starts server (production)
- `npm run dev:full` - **One-command setup**: Creates DB + tables + starts server (development)
- `npm run start:seed` - Same as `start:full` but also seeds demo data
- `npm run dev:seed` - Same as `dev:full` but also seeds demo data

### Database Commands
- `npm run migrate` - Create database and tables
- `npm run seed` - Populate database with demo data
- `npm run create-admin` - Create/update admin user
- `npm run setup` - Run migrate + seed (convenience script)

### Build Commands
- `npm run build:css` - Build TailwindCSS with watch mode
- `npm run build:css:prod` - Build TailwindCSS for production (minified)

## Pages

1. **Home** (`/`) - Hero section, trust badges, how it works, services grid, membership teaser, testimonials carousel, FAQ accordion, newsletter signup

2. **Services** (`/services`) - Filterable service categories with detailed service cards and safety & hygiene banner

3. **Pricing** (`/pricing`) - Three membership plans (Silver, Gold, Platinum), comparison table, and add-on services

4. **Booking** (`/booking`) - Multi-step booking form with service selection, pet details, address, and contact information

5. **About** (`/about`) - Brand story, why choose Stella, team profiles, service areas, working hours, and contact form

## Form Submissions

- **Booking Form** (`POST /booking`) - Stores submissions in memory and logs to console
- **Contact Form** (`POST /about/contact`) - Stores submissions in memory and logs to console

All form data is stored in `data/storage.js` and logged to the console for development purposes.

## Admin Module

The site includes a complete admin panel for managing content and bookings.

**Quick Start:**
1. Create admin user: `npm run init-admin admin yourpassword`
2. Access admin: `http://localhost:3000/admin/login`
3. Login with your credentials

See [ADMIN_README.md](./ADMIN_README.md) for complete admin documentation.

**Admin Features:**
- Secure authentication with bcrypt
- Dashboard with statistics
- Services CRUD
- Pricing plans CRUD
- Bookings management (view, filter, update status)
- Search and pagination

## Notes

- Placeholder images should be added to `public/images/`:
  - `favicon.ico`
  - `og-image.jpg` (1200x630px recommended)
  - `apple-touch-icon.png` (180x180px)
- Dark mode preference is saved in localStorage
- Form submissions are stored in JSON files (`data/bookings.json`)
- Admin data stored in JSON files (easily migratable to database)
- For production, set `SESSION_SECRET` environment variable
